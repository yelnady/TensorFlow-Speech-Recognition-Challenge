Introduction

This basic speech recognition AI was created with the TensorFlow Speech Recognition contest on Kaggle. It can recognize 30 different words. We created this to familiarize ourselves with the libraries and concepts used in the Auto Captioner. For this project, we used Keras Deep Learning Framework. 

The input should be a (.wav) file of one second. The 30 supported words are ( train, audio, off, six, up, dog, bird, bed, right, tree, zero, three, nine, five, two, happy, stop, no, down, go, cat, marvin, seven, four, eight, house, one, yes, wow, on, left, and sheila ).

Kaggle Competition

TnesorFlow Speech Recognition Challenge is a competition hosted on Kaggle by Google Brain to create an AI model that would recognize simple speech commands. Ten command words were given, as well as twenty other words.

We used their dataset to build an Deep Learning model from scratch that recognizes all thirty words from a short recording.

Kaggle Chall​enge​​: Project Pip​eline

1.

Acquiring Data

For this project, Kaggle provided a dataset for the challenge they created. The data consisted of several directories labeling the files, and in those directories the audio files in the .wav format.

2.

Processing Data

Working with a challenge dataset meant that the data, for the most part, was relatively clean. Since our project only worked with specific words from the dataset, we had to remove the data that didn't fit in our categories. This process also involved resampling the audio files from 16,000 samples per second to 8,000 samples per second.

3.

Modeling

Our project utilized a custom convolutional neural network in order to identify patterns in the wave data which allows the neural network to point out and identify what sounds make up a specific word.

Kaggle Challenge Model Structure

General Str​ucture

The main structure of our model can be identified by a few major parts:

  1. Four Convolution 1D Layers containing a MaxPooling1D and Dropout operation, with a ReLU activation function.
  2. Flatten Layer
  3. Two Dense Neural Network Layers, the first two layers utilizing ReLU and the final one utilizing SoftMax, which allows the classification of the audio file.

Training Information

The training information effects how the model runs and calculates outputs:

  • Categorical Crossentropy as our loss function.
  • EarlyStopping in order to avoid overfitting in our model.
  • Batch size of 32.
  • The number of epochs is usually around 25-35 until EarlyStopping kicks in.
  • The accuracy against validation data usually ends up around 85-90%.
  • The number of weights/parameters is around 800,000.

Dataset 

We used the dataset from the TensorFlow Speech Recognition Challenge by Google Brain on Kaggle. The dataset contained over ​64,727 audio files. These are various recordings of different speakers saying different words in each file.
